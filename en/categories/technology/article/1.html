Introduction to Java Programming

  • January 28, 2021
  • By Sastelvio Serafim Manuel

This article explains the history, fundamentals, and use of the Java programming language. It also contains examples for standard programming tasks.

This article focuses on the practical component but presents the essential concepts of technology as it progresses. The goal is to show simple code fragments that demonstrate a useful situation for using class, variables, and methods in Java.

Introduction

How was the Java Language conceived?

It is a programming language created in 1991 by James Gosling, Mike Sheridan and Patrick Naughton, a team of engineers at Sun Microsystems Inc known as the "Green Team". Sun Microsystems launched its first implementation publicly in 1996 as Java 1.0. The Java 1.0 compiler was rewritten in Java by Arthur Van Hoff to comply strictly with its specifications. With the arrival of Java 2.0, new versions had multiple configurations built for different types of platforms. In 1997, Sun Microsystems approached ISO and later formalized Java, but soon withdrew from the process. At the same time, Sun made most of its Java implementations available free, despite being a proprietary application. Sun generated revenue from Java by selling licenses for specialized products, such as the Java Enterprise System. On November 13, 2006, Sun released much of its Java Virtual Machine (JVM) as free open-source. On May 8, 2007, Sun completed the process by making all the main JVM code available under the terms of the open-source distribution.

Java Properties

The Java language was designed with the following properties:

  • Interpreted and compiled language:

    The Java source code is transformed into a bytecode that does not depend on the target platform, I mean, allows it to run on any device that processes data in bytes. These bytecode instructions will be interpreted by the Java Virtual machine (JVM). The JVM contains the so-called Hotspot-Compiler, which translates performance-critical bytecode instructions into native code instructions.

  • Object Oriented Programming Language:

    Object-Oriented Programming is a way of organizing application code as a collection of objects as an instance of a class. Concepts that will be deepened during this article and other articles on this blog.

    The four basic concepts of Object-Oriented Programming are:

    • Abstraction
    • Encapsulation
    • Inheritance
    • Polymorphism
  • Automatic memory management:

    Java manages the allocation and deallocation of memory when creating new objects. The application code doesn't have direct access to memory, however, the so-called Garbage Collector automatically deletes objects for which there is no active pointer.

  • Simple Language:

    Java is one of the simple languages, as it lacks complex features, such as pointers, operator overload, multiple inheritances, explicit memory allocation.

  • Robust Language:

    The Java language is robust, which means reliable. It is developed in order to place a large effort in checking errors as early as possible, so the java compiler can detect up to even those errors that are not easy to detect by another programming language. The main characteristics of Java that make it robust are Garbage Collector, exception handling, and memory allocation.

  • Secure Language:

    In java, we don't have pointers, so we can't access matrices outside the limit, that is, shows ArrayIndexOutOfBoundsException if we try to do so. Thatâs why several security holes, like stack corruption or buffer overflow, are impossible to exploit in Java.

  • Distributed Language:

    We can create distributed applications using the Java programming language. Remote Method Invocation and Enterprise Java Beans are used to create applications distributed in Java. Java programs can be easily distributed on one or more systems that are connected to each other through an Internet connection.

  • Multithreading:

    Java supports multithreading, which is the Java feature that allows running two or more parts of application using the best CPU performance.

  • Portable Language:

    As we know, Java code written on one machine can be executed on another device, even with different Operating Systems. The platform-independent feature of Java is possible due to the platform-independent bytecode and can run over all platforms.

The Java syntax is similar to C ++. Java is case sensitive, for instance, variables named myValue and myvalue are treated as different

Java Terminologies

Java Virtual Machine (JVM)

At run time, the Java Virtual Machine (JVM) reads and interprets .class files and executes the application instructions on the native hardware platform for which the JVM was written. The JVM interprets the bytecode to the Assembly language instructions making the CPU understand it. The difference is that the JVM is a piece of application written specifically for a given platform. The JVM is the core of the âwrite once, run anywhereâ principle of the Java language. The code can run on any chipset for which the proper JVM implementation is available. JVMs are available for major platforms, such as Linux, Windows and Mac, also for cell phones and other niche devices.

Java Development Kit (JDK)

When you download a Java Development Kit (JDK), you get, - in addition to the compiler and other tools, -a complete class library of pre-construction utilities that helps you perform most common application development tasks. Read the JDK Documentation for more details.

Java Runtime Environment (JRE)

The Java Runtime Environment (JRE) includes the JVM, code libraries, and components necessary to run programs that are written in the Java language. The JRE is available for several platforms. You can freely redistribute the JRE with your applications, according to the terms of the JRE license, to provide application users with a platform on which they run your application. The JRE is included in the JDK by default.

Java Compiler (JAVAC)

When writing application on the Java platform, the source code is stored in .java files, and then the compiler compiles them. The compiler checks your code against the language's syntax rules and then generates bytecode in .class files. Bytecode is a set of instructions designed to execute on a Java virtual machine (JVM). By including this level of abstraction, the Java compiler differs from other language compilers, that write appropriate instructions to the CPU chipset on which the program is run, thus allowing portability.

Garbage Collector

Instead of forcing the application to maintain memory allocation (or use a third-party library for this), the Java platform provides non-standard memory management in other languages. When your Java application creates an object instance at run time, the JVM automatically allocates memory space for that object from a heap memory pool - reserved for your application's use. The Java Garbage Collector runs in the background, keeping track of which objects the application no longer needs and reclaiming memory from them. This approach to memory manipulation is called implicit memory management because it does not require writing any memory manipulation code. Garbage collection is one of the essential resources for the performance of the Java platform.

Java Class

A class is a description of reality, I mean, a blueprint for a discrete entity (object) that contains attributes and behaviors. The class defines the basic structure of the object and, at run time, your application creates an instance of the object. An object has a defined limit and state and can take actions when duly requested. Each Object-Oriented Language has rules on how to define a class. 

//a basic Java program
import java.io.*; 
public class HelloWorld { 
    public static void main(String[] args) 
    { 
        System.out.println("Hello World"); 
    } 
}
Explanation:

  1. Comments: developers can include comments to help the documentation of the code. The Java syntax allows for single line and multiline comments. A single line comment must be contained in one line, although it is possible to use comments from an adjacent single line to form a block. A multiline comment starts with / *, must end with * /, and can cover any number of lines.

    Single line comments: 
    // Single line comment
    Multi-line comments: 
    /* 
Multi-line comment
This is another line that is included in the same comment
*/

  2. import java.io.*: it means that all classes in the io package can be imported to your class. The Java io package provides a set of input and output streams for reading and writing data to files or other sources of input or output.

  3. class: identifies that a class is being created that will contain attributes and methods. The HelloWorld class has only one main method in Java.

  4. static: the static keyword tells us that this method is accessible without instantiating the class.

  5. void: the void keyword tells you that this method will return nothing.

  6. main(): is the first method to be executed by the JVM in a Java program, that is, every Java application MUST have a main () method defined exactly as shown in the example, otherwise, the program will not execute.

  7. public: it means that the class, method, or attribute can be accessed by any other class or method that is inside or in another program to communicate with it.

  8. System.out: this is the standard output stream used to produce the result of a program on an output device such as a computer screen.

  9. println(): this method in Java is used to display text on the console. It prints the text on the console and the cursor moves to the beginning of the next line on the console. The next impression occurs from the next line.

Java Package

With the Java language, you can choose the names of your classes, such as Book, Animal or Person. You may end up using the same name to name two different classes. This situation is called a name collision and occurs frequently. The Java language uses packages to resolve these conflicts. A Java package is a mechanism that provides a namespace - from an area within which names are unique, but outside of them they may not be. To uniquely identify a building, it is necessary to qualify it including its namespace. Packages also provide you with a good way to create more complex applications with discrete units of functionality. 

package entity;

Import in Java

An import tells the Java compiler where to find classes referenced within the code. Any non-trivial class uses other classes for some functionality and importing is how the Java compiler is told about them. 

import entity.Car;

To import all classes within a package, just. * After the package name. Importing a package in its entirety can make the code less readable, so it is recommended to import only the necessary classes, using their names in full. 

import entity.*;

Java Naming Convention

Class

It is possible to name classes as the programmer wishes, but the convention is to use camel case: start with a capital letter, change it to concatenated capital letters and make all other letters lowercase. Class names must contain only letters and numbers. Adherence to these guidelines ensures that the code is more accessible to other developers who are following the same conventions.

A class can have essentially two types of members: variables and methods.

Variables

The values of variables of a specific class distinguish each instance of that class and define its state. These values are often referred to as instance variables. A variable has:

  • accessModifier (Access Modifier)
  • dataType (Stored Data Type)
  • variableName (Variable Name)
  • Opcionalmente, initialValue (Initialization Value)

The accessModifier can be defined as:

    public: any object in any program package can see the variable. 
    public String name;

    It is strongly recommended to avoid using public variables, but in extremely rare cases, they may be necessary, therefore, the option exists. The Java platform doesn't limit your use cases, so it's up to you to be disciplined about using good coding conventions, even when prompted to do otherwise.

    protected: any object defined in the same package or a subclass (defined in any package) can see the variable. 
    protected String name;
    private: only the class containing the variable can see it. 
    private String name;
    No specifier (also called friendly access or private package): only objects whose classes are defined in the same package can see the variable. 
    String name;

The dataType variable depends on what the variable is, it can be a primitive type or another type of class. 

//primitive
private String name;

//class
private Car car;

The variableName it is the programmer's responsibility to define the name, but, by convention, variable names use the camel case convention because they start with a lowercase letter.

The initialValue it is when a value is defined for the variable at the moment of the declaration. 

private String name = "Sastelvio";
Methods

A class's methods define its behavior. Methods fall into two main categories: constructors or all other methods, of which there are many types. A constructor method is used only to create an instance of a class. Other types of methods can be used for virtually any program behavior.

The definition of a method includes the following elements:

  • accessModifier (Access Modifier)
  • returnType (Return Type)
  • methodName (Method Name)
  • argumentList (Arguments)

The combination of these structural elements in a method definition is called a method signature.

The accessModifier follows the same specifications previously described in the variables.

The returnType informs the type of return that the method will give after executing the commands present in it. A method can return void (No Data), when it returns nothing at the end of its execution, primitive type when it returns a value of a type primitive or class, when it returns a class-type value. 

//void return method (No Data)
private void submit(){
}

//primitive type return method
private int count(){
}

//method with type return of a class
private Car accelerate(){
}

The argumentList are the arguments/parameters that the method uses to carry out its activities and are separated by commas and the arguments cannot have the same name in the same method, it can be argumentType is a primitive type or another type of class. 

//method with primitive data type argument
private void submit(int age, String name){
}

//method with class data type argument
private int count(Person person){
}
Constructor Method

Constructor methods are used to specify how to instantiate a class. The name of the constructor must match the name of the class. Therefore, if you call your class Person , the name of the builder must also be Person. And for any builder other than the default builder, there must be an argumentList

The entire Java class has a constructor method with no arguments that is used by default to instantiate the class. However, as soon as a constructor method with arguments/parameters is created, the standard constructor method is automatically disabled, making its explicit definition necessary. 

package entity;
public class Person {
        //variables
        private String name;
        private int age;
        private double height;
        private double  weight;
        private String eyeColor;
        private String gender;

        //constructor method without arguments
        public Person() {
                // acction when instantiate the non-argument method
        }

        //constructor method with arguments
        public Person(String name, int age, double height, double weight, String eyeColor, String gender) {
                this.name = name;
                this.age = age;
                this.height = height;
                this.weight = weight;
                this.eyeColor = eyeColor;
                this.gender = gender;
        }
}

In the code above there is the keyword this which allows informing the compiler that this is the class variable since in the arguments of the constructor method there are arguments with the same names.
