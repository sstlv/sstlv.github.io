Introdução à Programação em Java

  • 28 de Janeiro de 2021
  • Por Sastélvio Serafim Manuel

Este artigo explica a história, os fundamentos e o uso da linguagem de programação Java. Ele também contém exemplos para tarefas de programação padrão.

Este artigo tem o foco na componente prática, porém apresenta os conceitos essenciais da tecnologia à medida que vai avançando. O objetivo é mostrar fragmentos de códigos simples que demonstrem uma situação útil para utilizar classe, variáveis e métodos em Java.

Introdução

Como nasceu a Linguagem Java?

É uma linguagem de programação criada em 1991 por James Gosling, Mike Sheridan e Patrick Naughton, uma equipe de engenheiros da Sun Microsystems Inc conhecida como "Green team". A Sun Microsystems lançou sua primeira implementação pública em 1996 como Java 1.0. O compilador Java 1.0 foi reescrito em Java por Arthur Van Hoff para cumprir estritamente com suas especificações. Com a chegada do Java 2.0, novas versões tiveram múltiplas configurações construídas para diferentes tipos de plataformas. Em 1997, a Sun Microsystems abordou a ISO e depois formalizou o Java, mas logo se retirou do processo. Ao mesmo tempo, a Sun disponibilizou a maioria de suas implementações Java gratuitamente, apesar de seu status de software proprietário. A Sun gerou receita com Java por meio da venda de licenças para produtos especializados, como o Java Enterprise System. Em 13 de novembro de 2006, a Sun lançou grande parte de sua Máquina virtual Java (Java Virtual Machine) como software de código aberto gratuito. Em 8 de maio de 2007, a Sun concluiu o processo, disponibilizando todo o código principal da JVM sob os termos de distribuição de código aberto.

Propriedades da Linguagem Java

A linguagem Java foi projetada com as seguintes propriedades:

  • Linguagem interpretada e compilada:

    O código-fonte Java é transformado em bytecode que não depende da plataforma de destino, ou seja, permite que seja executado em qualquer maquina que processe dados em bytes. Essas instruções de bytecode serão interpretadas pela Java Virtual machine (JVM). A JVM contém o chamado Hotspot-Compiler, que traduz as instruções de bytecode críticas de desempenho em instruções de código nativo.

  • Linguagem de Programação Orientada a Objectos:

    A Programação Orientada a Objectos é uma maneira de organizar programas como uma coleção de objectos, cada um representando uma instância de uma classe. Conceitos que serão aprofundados no decorrer deste artigo e outros deste blog.

    Os quatro conceitos básicos da Programação Orientada a Objectos são:

    • Abstração
    • Encapsulamento
    • Herança
    • Polimorfismo
  • Gestão automática de memória:

    Java faz a gestão da alocação e desalocação de memória para a criação de novos objectos. O programa não tem acesso direto à memória, entretanto, o chamado coletor de lixo (Garbage Collector) exclui automaticamente objectos para os quais não existe nenhum ponteiro ativo.

  • Linguagem Simples:

    Java é uma das linguagens simples, pois não possui recursos complexos, como ponteiros, sobrecarga de operador, herança múltipla, alocação de memória explícita.

  • Linguagem Robusta:

    A linguagem Java é robusta, o que significa confiável. Ela é desenvolvida de forma a colocar um grande esforço na verificação de erros o mais cedo possível, por isso o compilador java é capaz de detectar até mesmo aqueles erros que não são fáceis de detectar por outra linguagem de programação. As principais características do Java que o tornam robusto são: coleta de lixo (Garbage Collector), manipulação de exceções e alocação de memória.

  • Linguagem Segura:

    Em java, não temos ponteiros e, portanto, não podemos acessar matrizes fora do limite, ou seja, mostra ArrayIndexOutOfBoundsException se tentarmos fazer isso. É por isso que várias falhas de segurança, como corrupção de pilha ou estouro de buffer, são impossíveis de explorar em Java.

  • Linguagem Distribuída:

    Podemos criar aplicativos distribuídos usando a linguagem de programação Java. Remote Method Invocation e Enterprise Java Beans são usados para criar aplicativos distribuídos em Java. Os programas Java podem ser facilmente distribuídos em um ou mais sistemas que estão conectados entre si por meio de uma conexão com a Internet.

  • Multithreading:

    Java oferece suporte a multithreading, que é um recurso Java que permite a execução simultânea de duas ou mais partes de um programa para utilização máxima da CPU.

  • Linguagem Portátil:

    Como sabemos, o código Java escrito em uma máquina pode ser executado em outra máquina, até com Sistemas Operactivos diferentes. O recurso independente de plataforma de Java é possível porque o bytecode independente de plataforma e pode ser levado para qualquer plataforma para execução.

A sintaxe Java é semelhante a C ++. Java diferencia maiúsculas de minúsculas, por exemplo, variáveis chamadas myValue e myvalue são tratadas como variáveis diferentes

Terminologias Java

Java Virtual Machine (JVM)

No tempo de execução, a Máquina Virtual Java (JVM) lê e interpreta arquivos .class e executa as instruções do programa na plataforma de hardware nativa para qual a JVM foi escrita. A JVM interpreta o bytecode como uma CPU interpretaria instruções de Linguagem Assembly. A diferença é que a JVM é uma parte do software escrita especificamente para uma determinada plataforma. A JVM é o núcleo do princípio “Único código, execução em qualquer plataforma” da linguagem Java. Seu código pode executar em qualquer chipset para o qual a implementação da JVM adequada está disponível. JVMs estão disponíveis para principais plataformas, como Linux, Windows Mac, e subconjuntos de linguagem Java foram implementados nas JVMs para telefones celulares e outros dispositivos de nicho.

Java Development Kit (JDK)

Ao fazer o download de um Java Development Kit (JDK), você obtém, — além do compilador e de outras ferramentas, — uma biblioteca de classe completa de utilitários de pré-construção que o ajuda a realizar tarefas de desenvolvimento de aplicativo mais comuns. Leia a Documentação do JDK para mais detalhes.

Java Runtime Environment (JRE)

O Java Runtime Environment (JRE) inclui a JVM, bibliotecas de códigos e componentes necessários para executar programas que são escritos na linguagem Java. O JRE está disponível para diversas plataformas. É possível redistribuir livremente o JRE com seus aplicativos, de acordo com os termos da licença do JRE, para fornecer aos usuários do aplicativo uma plataforma na qual executar seu software. O JRE está incluído no JDK por padrão.

Compilador Java (JAVAC)

Quando se escreve um programa na plataforma Java, escreve seu código-fonte em arquivos .java e depois os compila. O compilador verifica seu código nas regras de síntaxe da linguagem e depois gera bytecode em arquivos .class. Bytecode é um conjunto de instruções destinadas a executar em uma Java virtual machine (JVM). Ao incluir esse nível de abstração, o compilador Java difere-se de outros compiladores de linguagem, que escrevem instruções adequadas para o chipset de CPU no qual o programa é executado, permitindo assim a portabilidade.

Garbage Collector

Ao invés de forçá o programa a manter a alocação de memória (ou usar uma biblioteca de terceiros para isso), a plataforma Java fornece gestão de memória fora do padrão das outras linguagens. Quando seu programa Java cria uma instância de objecto no tempo de execução, a JVM aloca automaticamente espaço de memória para esse objecto a partir de um conjunto de memória heap — reservado para uso de seu programa. O Garbage Collector (coletor de lixo) Java é executado em segundo plano, mantendo o controle de quais objectos o aplicativo não necessita mais e recuperando memória deles. Essa abordagem para manipulação de memória é chamada de gestão implícita de memória porque não exige a escrita de qualquer código de manipulação de memória. A coleta de lixo é um dos recursos essenciais para o desempenho da plataforma Java.

Palavras Reservadas

Como qualquer linguagem de programação, a linguagem Java designa determinadas palavras que o compilador reconhece como especiais. Por essa razão, não podem ser usadas para nomear suas construções Java. A lista de palavras reservadas é surpreendentemente curta:

abstract assert boolean
break byte case
catch char class
const continue default
do double else
enum extends final
finally assert float
goto if implements
import instanceof int
interface long native
new para package
private protected public
return short static
strictfp super switch
synchronized this throw
throws transient try
void volatile while

Observe que true, false e null são tecnicamente palavras não reservadas. Embora sejam literais, também não pode usá-las para nomear construções Java.

Uma vantagem da programação com um IDE é que ela pode usar a coloração de síntaxe para palavras reservadas

Classe Java

Uma classe é uma descrição de uma realidade, ou seja, um blueprint para uma entidade discreta (objecto) que contém atributos e comportamentos. A classe define a estrutura básica do objecto e, no tempo de execução, seu programa cria uma instância do objecto. Um objecto tem um limite e um estado definidos e pode fazer ações quando devidamente solicitado. Cada Linguagem Orientada a Objectos possui regras sobre como definir uma classe. 

//um programa básico em Java
import java.io.*; 
public class HelloWorld { 
    public static void main(String[] args) 
    { 
        System.out.println("Hello World"); 
    } 
}
Explicação:

  1. Comentários: os programadores podem incluir comentários para ajudar a documentar o código. A sintaxe Java permite comentários de única linha e multilinhas. Um comentário de única linha deve estar contido em uma linha, embora seja possível usar comentários de única linha adjacentes para formar um bloco. Um comentário de multilinhas começa com /*, deve terminar com */ e pode abranger qualquer número de linhas.

    Comentários de linha única: 
    // Comentário de linha única
    Comentários de multiplas linhas: 
    /* 
Comentário de multiplas linhas 
Esta é outra linha que esta inclusa no mesmo comentário
*/

  2. import java.io.*: significa que todas as classes do pacote io podem ser importadas. O pacote Java io fornece um conjunto de fluxos de entrada e saída para leitura e escrita de dados em ficheiros ou outras fontes de entrada ou saída.

  3. class: identifica que esta a ser criada uma classe que vai conter atributos e métodos. A classe HelloWorld possui apenas um método main em Java.

  4. static: a palavra-chave static nos diz que este método é acessível sem instanciar a classe.

  5. void: a palavras-chave void informa que este método não retornará nada.

  6. main(): é o primeiro método a ser executado pela JVM num programa Java, ou seja, todo programa Java DEVE ter um metodo main() definido exactamente do jeito mostrado no exemplo, caso contrário, o programa nao executa.

  7. public: significa que a classe,método ou atributo poder ser acedido por qualquer outra classe ou método que esteja dentro ou num outro programa a comunicar com este.

  8. System.out: este é o fluxo de saída padrão usado para produzir o resultado de um programa em um dispositivo de saída como a tela do computador.

  9. println(): este método em Java é usado para exibir um texto no console. Ele imprime o texto no console e o cursor se move para o início da próxima linha no console. A próxima impressão ocorre a partir da próxima linha.

Pacote em Java

Com a linguagem Java, é possível escolher os nomes de suas classes, como Livro, Animal ou Pessoa. Pode ocorrer de acabar usando o mesmo nome para denominar duas classes diferentes. Esta situação é chamada de colisão de nomes e ocorre com frequência. A linguagem Java usa pacotes para resolver esses conflitos. Um pacote Java é um mecanismo que fornece um namespace - de uma área dentro da qual nomes são exclusivos, mas fora dela eles podem não ser-. Para identificar uma construção exclusivamente, é necessário qualificá-la incluindo seu namespace. Pacotes também fornecem a você uma boa forma de criar aplicativos mais complexos com unidades de funcionalidade discretas. 

package entidade;

Importação em Java

Uma importação informa ao compilador Java onde localizar classes às quais se fez referência dentro do código. Qualquer classe não trivial usa outras classes para alguma funcionalidade e a importação é como se informa ao compilador Java sobre elas. 

import entidade.Carro;

Para importar todas as classes dentro de um pacote, basta .* após o nome do pacote. A importação de um pacote na sua totalidade pode tornar o código menos legível, portanto, é recomendado importar apenas as classes necessárias, usando seus nomes integralmente. 

import entidade.*;

Convenção de Nomenclatura em Java

Classes

É possível nomear classes como o programador desejar, mas a convenção é usar camel case: comece com uma letra maiúscula, altere para letras maiúsculas concatenada e torne todas as outras letras minúsculas. Nomes de classes devem conter apenas letras e números. A adesão destas diretrizes garante que o código seja mais acessível a outros desenvolvedores que estão seguindo as mesmas convenções.

Uma classes podem ter essencialmente dois tipos de membros: variáveis e métodos.

Variáveis

Os valores de variáveis de uma classe específica distinguem cada instância daquela classe e definem seu estado. Esses valores geralmente são referidos como variáveis de instância. Uma variável possui:

  • accessModifier (Modificador de Acesso)
  • dataType (Tipo de Dado armazenado)
  • variableName (Nome da Variável)
  • Opcionalmente, initialValue (Valores de Inicialização)

O accessModifier pode ser definido como:

    public: qualquer objecto em qualquer pacote do programa pode ver a variável. 
    public String nome;

    É fortemente recomendado evitar o uso de variáveis públicas, mas em casos extremamente raros, elas podem ser necessárias, portanto, existe a opção. A plataforma Java não limita seus casos de uso, portanto cabe a você ser disciplinado sobre o uso de boas convenções de codificação, mesmo quando instigado a fazer de outra forma.

    protected: qualquer objecto definido no mesmo pacote ou uma subclasse (definida em qualquer pacote) pode ver a variável. 
    protected String nome;
    private: apenas a classe contendo a variável pode vê-la. 
    private String nome;
    Nenhum especificador (também chamado de acesso amigável ou pacote privado): apenas objectos cujas classes são definidas no mesmo pacote podem ver a variável. 
    String nome;

O dataType de variável depende do que a variável é, ela pode ser um tipo primitivo ou outro tipo de classe. 

//primitivo
private String nome;

//classe
private Carro carro;

O variableName é responsabilidade do programador definir o nome, mas, por convenção, nomes de variáveis usam a convenção de camel case pelo fato de começarem com uma letra minúscula.

O initialValue é quando se define um valor para a variavel no momento da declaração. 

private String nome = "Sastelvio";
Métodos

Os métodos de uma classe definem seu comportamento. Métodos se enquadram em duas categorias principais: construtores ou todos os outros métodos, dos quais existem muitos tipos. Um método construtor é usado apenas para criar uma instância de uma classe. Outros tipos de métodos podem ser usados virtualmente para qualquer comportamento do programa.

A definição de um método inclui os seguintes elementos:

  • accessModifier (Modificador de Acesso)
  • returnType (Tipo de Retorno)
  • methodName (Nome do Método)
  • argumentList (Argumentos)

A combinação desses elementos estruturais em uma definição de método é chamada de assinatura de método.

O accessModifier segue as mesmas especificações descritas anteriormente nas variáveis.

O returnType informa o tipo de retorno que o método vai dar depois de executar os comandos nele presentes. Um método pode ter retorno void (Vazio), quando não retorna nada ao fim de sua execução, retorno de tipo primitivo quando retorna um valor de um tipo primitivo ou retorno de classe, quando retorna um valor do tipo de uma classe. 

//método com retorno void (Vazio)
private void submeter(){
}

//método com retorno de tipo primitivo
private int contar(){
}

//método com retorno do tipo de uma classe
private Carro acelerar(){
}

O argumentList são os argumentos/parámetros que o método usa para realizar a sua acção e são separados por vírgulas e os argumentos não pode ter o mesmo nome no mesmo método, pode ser argumentType é um tipo primitivo ou outro tipo de classe. 

//método com argumento de tipo primitivo
private void submeter(int idade, String nome){
}

//método com argumento de tipo de classe
private int contar(Pessoa pessoa){
}
Método Construtor

Métodos construtores são usados para especificar como instanciar uma classe. O nome do construtor deve corresponder ao nome da classe. Portanto, se você chamar sua classe de Pessoa, o nome do construtor também deverá ser Pessoa. E para qualquer construtor que não seja o construtor padrão, deve ter um argumentList

Toda a classe Java tem um método construtor sem argumentos que é usado por padrão para instanciar a classe. No entanto, a partir do momento em que se cria um método construtor com argumentos/parámetros, automaticamente o método construtor padrão fica desactivado, tornando necessária a sua definição explicita. 

package entidade;
public class Pessoa {
        //variáveis
        private String nome;
        private int idade;
        private double altura;
        private double  peso;
        private String corOlhos;
        private String genero;

        //método construtor sem argumentos
        public Pessoa() {
                // acção ao instanciar o método com o construtor sem argumentos
        }

        //método construtor com argumentos
        public Person(String nome, int idade, double altura, double peso, String corOlhos, String genero) {
                this.nome = nome;
                this.idade = idade;
                this.altura = altura;
                this.peso = peso;
                this.corOlhos = corOlhos;
                this.genero = genero;
        }
}

No código acima existe a palavra-chave this que permite informar ao compilador que se trata da variável da classe visto que nos arumentos do método construtor existem argumentos com os mesmos nomes.

Tipos de Dados Primitivos

Os tipos de dados são uma combinação de valores e de operações que uma variável pode executar, o que pode variar conforme o sistema operacional e a linguagem de computador. São utilizados para indicar ao compilador as conversões necessárias para obter os valores em memória durante a compilação do programa.

Em Java, temos:

Tipo Tamanho Valor Padrão Intervalo de Valores
boolean n/a false true ou false
byte 8 bits 0 -128 a 127
char 16 bits n/a \u0000′ \u0000′ a \uffff’ ou 0 a 65535
short 16 bits 0 -32768 a 32767
int 32 bits 0 -2147483648 a 2147483647
long 64 bits 0 -9223372036854775808 a 9223372036854775807
float 32 bits 0.0 1.17549435e-38 a 3.4028235e+38
double 64 bits 0.0 4.9e-324 a 1.7976931348623157e+308

Comumente o String é identificado como um tipo primitivo,entretanto, ele não é. É um tipo especial de dados que deriva da classe java.lang.Object, que de uma forma grosseira, pode ser identificado como um Array de chars.
